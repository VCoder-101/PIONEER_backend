# Реализация системы управления расписанием и слотами

## Обзор

Реализована полноценная система управления временем работы организаций и доступными слотами для записи клиентов.

## Компоненты системы

### 1. Модели данных

#### OrganizationSchedule
Расписание работы организации по дням недели.

**Поля:**
- `organization` - связь с организацией
- `weekday` - день недели (0-6, где 0 = понедельник)
- `is_working_day` - рабочий день (boolean)
- `open_time` - время открытия
- `close_time` - время закрытия
- `break_start` - начало перерыва (опционально)
- `break_end` - конец перерыва (опционально)
- `slot_duration` - длительность слота в минутах (5-240)
- `is_active` - активность записи

**Особенности:**
- Уникальная комбинация `organization` + `weekday`
- Валидация времени в сериализаторе
- Индексы для быстрого поиска

#### OrganizationHoliday
Выходные и праздничные дни организации.

**Поля:**
- `organization` - связь с организацией
- `date` - дата выходного
- `reason` - причина (опционально)
- `is_active` - активность записи

**Особенности:**
- Уникальная комбинация `organization` + `date`
- Индексы по дате для быстрого поиска

#### ServiceAvailability
Специфичное расписание для отдельных услуг (переопределяет общее расписание организации).

**Поля:**
- `service` - связь с услугой
- `weekday` - день недели (0-6)
- `available_from` - время начала доступности
- `available_to` - время окончания доступности
- `max_bookings_per_slot` - максимальное количество записей на слот (1-10)
- `is_active` - активность записи

**Особенности:**
- Уникальная комбинация `service` + `weekday`
- Позволяет настроить параллельное обслуживание (несколько клиентов одновременно)

### 2. API Endpoints

#### Управление расписанием
- `GET /api/organizations/schedules/` - список расписаний
- `POST /api/organizations/schedules/` - создание расписания
- `GET /api/organizations/schedules/{id}/` - детали расписания
- `PATCH /api/organizations/schedules/{id}/` - обновление расписания
- `DELETE /api/organizations/schedules/{id}/` - удаление расписания

#### Управление выходными
- `GET /api/organizations/holidays/` - список выходных
- `POST /api/organizations/holidays/` - создание выходного
- `GET /api/organizations/holidays/{id}/` - детали выходного
- `PATCH /api/organizations/holidays/{id}/` - обновление выходного
- `DELETE /api/organizations/holidays/{id}/` - удаление выходного

#### Управление доступностью услуг
- `GET /api/organizations/service-availability/` - список правил
- `POST /api/organizations/service-availability/` - создание правила
- `GET /api/organizations/service-availability/{id}/` - детали правила
- `PATCH /api/organizations/service-availability/{id}/` - обновление правила
- `DELETE /api/organizations/service-availability/{id}/` - удаление правила

#### Получение доступных слотов
- `GET /api/organizations/available-slots/for_service/?service_id=X&date=YYYY-MM-DD` - список доступных слотов

### 3. Логика генерации слотов

Метод `_generate_slots` в `AvailableSlotsViewSet` реализует следующую логику:

1. **Проверка выходного дня**
   - Запрос к `OrganizationHoliday`
   - Если найден активный выходной - возвращаем пустой список

2. **Получение расписания организации**
   - Запрос к `OrganizationSchedule` по `organization` + `weekday`
   - Если нет расписания или `is_working_day=False` - возвращаем пустой список

3. **Проверка специфичного расписания услуги**
   - Запрос к `ServiceAvailability` по `service` + `weekday`
   - Если найдено - используем его параметры
   - Иначе - используем параметры из расписания организации

4. **Генерация слотов**
   - Начинаем с `start_time` (из расписания услуги или организации)
   - Заканчиваем `end_time` (с учетом длительности услуги)
   - Шаг - `slot_duration` из расписания организации
   - Пропускаем слоты в перерыве

5. **Учет текущего времени**
   - Если дата = сегодня, начинаем с `now + 1 час`
   - Округляем до ближайшего слота

6. **Проверка занятости**
   - Запрос к `Booking` для каждого слота
   - Считаем активные бронирования (`NEW`, `CONFIRMED`)
   - Сравниваем с `max_bookings_per_slot`

7. **Формирование ответа**
   - Для каждого слота возвращаем:
     - `time` - время в формате HH:MM
     - `datetime` - полная дата-время в ISO формате
     - `available` - доступен ли слот
     - `booked` - количество существующих бронирований
     - `capacity` - максимальная вместимость

### 4. Валидация при создании бронирования

В `BookingSerializer.validate()` добавлена комплексная валидация:

1. **Проверка времени**
   - Не в прошлом
   - Минимум 1 час от текущего момента

2. **Проверка выходного дня**
   - Запрос к `OrganizationHoliday`

3. **Проверка расписания**
   - Запрос к `OrganizationSchedule`
   - Проверка `is_working_day`

4. **Проверка рабочего времени**
   - Время в диапазоне `open_time` - `close_time`
   - Не попадает на перерыв

5. **Проверка доступности услуги**
   - Если есть `ServiceAvailability` - проверяем её диапазон

6. **Проверка занятости слота**
   - Запрос к `Booking` с фильтром по времени и статусу
   - Сравнение с `max_bookings_per_slot`

При любой ошибке возвращается понятное сообщение на русском языке.

### 5. Права доступа

#### OrganizationScheduleViewSet
- **ADMIN**: полный доступ ко всем расписаниям
- **CLIENT (владелец)**: доступ к расписаниям своих организаций
- **CLIENT (обычный)**: нет доступа

#### OrganizationHolidayViewSet
- **ADMIN**: полный доступ ко всем выходным
- **CLIENT (владелец)**: доступ к выходным своих организаций
- **CLIENT (обычный)**: нет доступа

#### ServiceAvailabilityViewSet
- **ADMIN**: полный доступ ко всем правилам
- **CLIENT (владелец)**: доступ к правилам услуг своих организаций
- **CLIENT (обычный)**: нет доступа

#### AvailableSlotsViewSet
- Все авторизованные пользователи могут просматривать доступные слоты

## Примеры использования

### Создание расписания для организации

```python
# Понедельник-пятница: 9:00-18:00 с перерывом 13:00-14:00
for weekday in range(5):
    OrganizationSchedule.objects.create(
        organization=org,
        weekday=weekday,
        is_working_day=True,
        open_time='09:00',
        close_time='18:00',
        break_start='13:00',
        break_end='14:00',
        slot_duration=30
    )

# Суббота: 10:00-16:00 без перерыва
OrganizationSchedule.objects.create(
    organization=org,
    weekday=5,
    is_working_day=True,
    open_time='10:00',
    close_time='16:00',
    slot_duration=30
)

# Воскресенье: выходной
OrganizationSchedule.objects.create(
    organization=org,
    weekday=6,
    is_working_day=False,
    open_time='00:00',
    close_time='00:00',
    slot_duration=30
)
```

### Добавление праздничных дней

```python
OrganizationHoliday.objects.create(
    organization=org,
    date='2026-05-01',
    reason='Праздник труда'
)
```

### Настройка специального расписания для услуги

```python
# Шиномонтаж доступен только 10:00-17:00, можно записать 2 клиентов одновременно
ServiceAvailability.objects.create(
    service=service,
    weekday=0,  # Понедельник
    available_from='10:00',
    available_to='17:00',
    max_bookings_per_slot=2
)
```

## Производительность

### Оптимизации
1. **Индексы БД**
   - `organization` + `weekday` для расписаний
   - `organization` + `date` для выходных
   - `service` + `weekday` для доступности услуг

2. **Запросы**
   - Используется `.first()` вместо `.get()` где возможно
   - Минимальное количество запросов в цикле генерации слотов

3. **Кэширование**
   - Расписание и выходные запрашиваются один раз
   - Бронирования проверяются одним запросом с фильтром
