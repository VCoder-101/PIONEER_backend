# Реализация функционала отмены бронирований и календарного формата

## Что реализовано

### 1. Специальный endpoint для отмены бронирований
- Добавлен `POST /api/bookings/{id}/cancel/` для отмены бронирований
- Реализована проверка прав доступа:
  - Клиент может отменить своё бронирование
  - Владелец организации может отменить бронирование на свои услуги
  - Администратор может отменить любое бронирование
- Добавлена валидация статуса:
  - Нельзя отменить уже отмененное бронирование (CANCELLED)
  - Нельзя отменить завершенное бронирование (DONE)
- Логирование, кто отменил бронирование (client/organization/admin)

### 2. Календарный формат данных (invoices)
- Добавлен `GET /api/bookings/calendar/` для получения бронирований в упрощенном формате
- Формат ответа соответствует требованиям фронтенда:
  ```json
  {
    "id": 0,
    "customerName": "Имя",
    "dateTime": "10/01/2026 10:30",
    "carModel": "Lada Vesta",
    "serviceMethod": "Комплекс",
    "duration": "60",
    "price": "600.00"
  }
  ```
- Поддержка фильтрации, поиска и пагинации
- Учет прав доступа (клиент видит свои, организация - свои услуги)

### 3. Обработка ошибок
- 403 Forbidden - нет прав на отмену
- 400 Bad Request - уже отменено
- 400 Bad Request - завершено (нельзя отменить)

## API готов к использованию

Функционал полностью реализован и протестирован. API готов к интеграции с фронтендом.

### Основные endpoints:
- `GET /api/bookings/` - список бронирований
- `GET /api/bookings/calendar/` - календарный формат
- `POST /api/bookings/{id}/cancel/` - отмена бронирования
- `POST /api/bookings/` - создание бронирования

Все endpoints поддерживают фильтрацию, поиск, сортировку и учитывают права доступа.
