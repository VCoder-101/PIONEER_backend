# БАГ-010: Создание услуги пользователем с организацией

## Описание проблемы
При попытке создать услугу пользователь с организацией получал ошибку 403 Forbidden:
```json
{
  "code": "permission_denied",
  "detail": "You do not have permission to perform this action.",
  "error": "Доступ запрещен"
}
```

Проблема возникала даже когда пользователь был владельцем организации.

## Причина
В `services/permissions.py` метод `has_permission` проверял только наличие организаций у пользователя через `request.user.organizations.exists()`, но не проверял статус организации. Пользователь мог иметь организацию со статусом `pending` или `rejected`, что не давало ему права создавать услуги.

Согласно бизнес-логике, пользователь может создавать услуги только для организаций со статусом `approved`.

## Решение

### 1. Обновлен `services/permissions.py`
Добавлена проверка статуса организации - теперь пользователь может создавать услуги только если у него есть хотя бы одна организация со статусом `approved`:

```python
def has_permission(self, request, view):
    # ...
    # Владелец организации (CLIENT с одобренной организацией) может создавать и управлять
    if hasattr(request.user, 'organizations'):
        # Проверяем наличие хотя бы одной одобренной организации
        approved_orgs = request.user.organizations.filter(organization_status='approved')
        if approved_orgs.exists():
            return True
    # ...
```

### 2. Обновлен `services/views.py`
Добавлена дополнительная проверка в `perform_create` - проверяется, что организация, для которой создается услуга, имеет статус `approved`:

```python
def perform_create(self, serializer):
    # ...
    # Проверяем, что организация одобрена
    if org.organization_status != 'approved':
        raise PermissionDenied("You can create services only for approved organizations")
    
    serializer.save()
```

## Тестирование
Создан тестовый скрипт `test_bug_010_service_create.py`, который:
1. Авторизуется под новым пользователем
2. Создает организацию (статус автоматически `pending`)
3. Проверяет, что нельзя создать услугу для неодобренной организации ✅
4. Одобряет организацию через админа (если админ доступен)
5. Создает услугу для одобренной организации

## Результат
Теперь пользователи могут создавать услуги только для одобренных организаций:
- Если у пользователя нет ни одной одобренной организации → 403 Forbidden
- Если пользователь пытается создать услугу для организации со статусом `pending` или `rejected` → 403 Forbidden с сообщением "You can create services only for approved organizations"
- Если у пользователя есть одобренная организация и он создает услугу для неё → 201 Created
